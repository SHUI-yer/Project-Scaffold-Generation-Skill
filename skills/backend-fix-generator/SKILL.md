# backend-fix-generator

## 触发条件

- 接口报错/返回不对/业务逻辑不对/鉴权无效/校验未生效
- `project-fix-generator` 将主归类判定为后端问题

## 小白模式

- 术语首次出现必须解释：Controller（接口入口）、Service（业务逻辑）、Repository（查数据库）、事务（要么都成功要么都失败）
- 给方案时说明“哪一层出问题、为什么、怎么验收”

## 最短流程

1) 先定位在哪一层（选一类为主）
- 接口入口（路由/方法/参数/返回结构）
- 业务逻辑（规则/状态流转/事务边界）
- 数据访问（查询条件/分页/映射/性能）
- 异常与校验（DTO 校验/错误码/全局异常）
- 鉴权与权限（JWT/Session/RBAC/接口保护）
- **安全响应规范**：认证失败必须返回 401 + JSON，权限不足返回 403 + JSON，不得返回 HTML 错误页

2) 每轮最多 4 题缩小范围（**必须使用 AskUserQuestion 工具以选项形式呈现，禁止要求用户手动打字**）

3) 输出“后端修改方案卡”
- 受影响接口清单
- 受影响模块/文件清单
- 风险点（兼容性/数据变更）
- 验收点（接口请求/响应示例、错误码、权限效果）

4) 修改后交付（产出优先）
- 文件清单
- 启动方式
- 验收点（curl/Postman 请求 + 期望返回）

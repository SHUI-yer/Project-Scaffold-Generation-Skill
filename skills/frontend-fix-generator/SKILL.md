# frontend-fix-generator

## 触发条件

- 页面缺失/页面不对/交互不顺/路由跳转异常/前端报错/接口联调不通
- `project-fix-generator` 将主归类判定为前端问题

## 小白模式

- 术语首次出现必须解释：路由（页面跳转规则）、联调（前后端对接）、TypeScript 类型（数据结构说明）
- 给方案时说明“改哪里、影响什么、如何验收”

## 最短流程

1) 先归类（选一类为主）
- 页面结构（缺页/布局错/组件拆分不合理）
- 交互流程（表单提交、弹窗、加载/错误态）
- 路由导航（路径、守卫、菜单映射）
- 类型与接口联调（请求/响应结构不一致、字段命名不一致）
- 状态与性能（状态丢失、重复请求、卡顿）

2) 每轮最多 4 题缩小范围（**必须使用 AskUserQuestion 工具以选项形式呈现，禁止要求用户手动打字**）

3) 输出“前端修改方案卡”
- 目标页面/组件清单
- 目标接口清单
- 风险点（影响范围）
- 验收点（如何判断已修复）

4) 修改后交付（产出优先）
- 文件清单
- 启动方式（dev server / build）
- 验收点（页面路径、操作步骤、期望结果）

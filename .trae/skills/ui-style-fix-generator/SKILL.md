# UI 风格专项修改生成器（UI Style Fix Generator）

## 目标

当问题集中在界面视觉与设计系统层时，围绕设计规范、主题模式、配色方案、布局模式、字体、动画、组件一致性和暗黑模式适配进行专项问询与增量修改。

## 何时调用

- 用户反馈“页面不好看”“风格不统一”“不是我要的主题/布局/配色”
- `project-fix-generator` 将主归类识别为 `UI 风格类`
- 功能基本可用，但视觉表达与设计目标不一致

## 小白友好沟通规范

- 若用户不理解“设计规范、主题、token、响应式、暗黑模式”这些词，首次出现时必须解释
- 解释视觉问题时，优先用“看起来像什么、哪里不统一、用户会感受到什么”来描述
- 比较不同风格时，要明确说出“更稳重/更活泼/更适合后台/更适合官网”这类直观差异
- 给推荐策略时，必须说明为什么推荐某种配色、布局或动画风格
- 尽量避免只说抽象设计词，不解释落地效果

## 核心原则

- 先识别偏差落在规范、颜色、布局、字体、动画还是组件一致性
- 优先通过设计变量和主题系统修正，而不是逐页面硬改样式
- 保持亮色、暗色、响应式和组件状态的一致性
- 样式修改不能破坏既有功能可用性
- 若用户是小白，默认补充“术语解释 + 风格对比 + 推荐理由”

## 专项问题分类

1) 设计规范类
- Material / Ant / Tailwind / Bootstrap 风格不一致
- 组件观感不统一
- 间距、圆角、阴影体系混乱
- 信息层级不清

2) 主题与配色类
- 主色不对
- 色阶不统一
- 状态色缺失
- 暗黑模式适配不完整

3) 布局与导航类
- 侧边栏/顶部导航不符合预期
- 内容区结构混乱
- 响应式断点不合理
- 单页与多页布局不统一

4) 字体与动效类
- 字体风格不匹配
- 字重与字号层级乱
- 动画过多或过少
- 过渡和反馈不统一

## 修改顺序

1. 对齐目标设计规范
2. 确认主题模式与配色策略
3. 统一全局 CSS 变量和设计 token
4. 修复导航与布局
5. 修复组件样式、字体、动效与暗黑模式

## 回归检查

- 设计规范一致
- 颜色、状态色、暗色模式完整
- 布局与导航符合预期
- 字体和字号层级统一
- 动效节奏与反馈一致
- 全局变量优先于页面硬编码样式

## 输出要求

- 输出 UI 偏差归类卡
- 输出受影响页面与样式层清单
- 输出主题/变量调整说明
- 输出视觉一致性回归摘要

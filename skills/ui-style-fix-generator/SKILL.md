# ui-style-fix-generator

## 触发条件

- 风格不统一/主题不对/配色不对/布局不对/暗黑模式不生效/动效不合适
- `project-fix-generator` 将主归类判定为 UI 风格问题

## 小白模式

- 术语首次出现必须解释：主题（全站颜色方案）、暗黑模式（深色主题）、设计规范（组件的统一风格）
- 给方案时说明“改动会影响哪些页面、怎么验收一致性”

## 最短流程

1) 先归类（选一类为主）
- 设计规范（Material/Ant/Tailwind/Bootstrap 或自定义）
- 主题与配色（主色、色阶、状态色、亮/暗模式）
- 布局与导航（侧边栏/顶部/混合/单页）
- 字体与动效（字体方案、动效强度、交互反馈）

2) 每轮最多 4 题缩小范围（选项式）

3) 输出“UI 修改方案卡”
- 目标规范与主题模式
- CSS 变量与 token 变更清单
- 受影响页面/组件范围
- 验收点（对比前后截图、暗黑模式切换、组件状态一致）

4) 修改后交付（产出优先）
- 文件清单（全局样式、主题变量、组件样式）
- 验收点（亮/暗、响应式、常见组件状态）
